# Приватность и безопасность

Это медицинский проект — он работает с записями консультаций и персональными данными. Поэтому к секретам и приватности здесь относятся строго.

---

## Чего НЕТ в этом репозитории

Публичная версия очищена. В git **не попали** и не должны попадать:

- ключи STT-сервиса, LLM (`llm_api_key`), `api_shared_key`;
- токен Telegram-бота и параметры прокси;
- реальные **ФИО врачей** (в промпте — условные имена) и любые данные пациентов;
- аудиозаписи и сгенерированные `.docx`;
- IP-адреса, домены и реквизиты инфраструктуры;
- личные Telegram-идентификаторы (chat_id, username).

В репозитории — только `config.example.json` с заглушками.

---

## Как устроены секреты

| Что | Где хранится | В репозитории |
|-----|--------------|---------------|
| Ключи STT / LLM / api_shared_key | `config.json` | ❌ (только пример) |
| Токен бота, прокси | `config.json` | ❌ |
| Ключи для systemd-сервисов | `EnvironmentFile` (вне репозитория) | ❌ |

`config.json`, `.env`, `*.env` — все в [`.gitignore`](.gitignore).

### Настройка

```bash
cd server
cp config.example.json config.json
# впишите свои значения вместо YOUR_*-заглушек
```

---

## Защита в коде

- **FastAPI-приёмник** требует `Authorization: Bearer <api_shared_key>` на каждый запрос загрузки.
- **Telegram-бот** обращается к API Telegram через прокси (при необходимости), токен — из конфига.
- **Прямые вызовы LLM** идут на HTTP-эндпоинт провайдера (`llm_base_url`) с ключом из окружения.

---

## Обращение с данными пациентов

- **Авто-очистка** (`cleanup.sh` по таймеру): входные аудио, временные файлы и готовые документы удаляются по истечении короткого срока — данные не накапливаются на диске.
- **Ротация логов** — логи усекаются, чтобы не разрастались и не хранили лишнего.
- Обработка идёт по одному файлу; промежуточные артефакты живут минимально необходимое время.

---

## Рекомендации при развёртывании

1. Никогда не коммитьте `config.json` с реальными ключами.
2. Для systemd используйте `EnvironmentFile` с правами `600` и владельцем сервисного пользователя.
3. FastAPI на `:8000` держите за TLS-прокси или во внутренней сети — не публикуйте «голым» в интернет.
4. Регулярно ротируйте `api_shared_key` и токен бота.
5. Соблюдайте требования к обработке медицинских персональных данных вашей юрисдикции.
